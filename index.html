Setup Instructions

1

Discord Bot Setup

Go to Discord Developer Portal, create a new application, and add a bot. Enable these Privileged Gateway Intents: Presence Intent, Server Members Intent, and Message Content Intent. Copy your bot token.

2

Bot Installation

Download the bot files and create a .env file with your DISCORD_TOKEN. Install dependencies: pip install discord.py python-dotenv PyNaCl. Optionally create configuration files from the examples.

3

Invite Bot to Server

Generate an invite URL with required permissions: View Channels, Send Messages, Embed Links, Attach Files, Read Message History, and View Audit Log. Required scopes: bot and applications.commands.

4

Configure Log Channel

Run the bot with python main.py, then use /setlogchannel command in your Discord server (admin only) to select where events will be logged. Each server can have its own independent log channel!

Multi-Guild Support

The bot fully supports multiple Discord servers! Each server can have its own dedicated log channel, configured independently using /setlogchannel.

Usage Guide

Commands

/setlogchannel <channel>

Set the log channel for event logging (Admin only). Select the channel where you want events to be logged for this specific server.

Running the Bot

Start the bot by running python main.py. The bot will display its username, ID, Discord.py version, and current log channel (if configured).

Configuration

The bot supports modular configuration via a config/ directory with separate files for colors, guilds, and toggles. Alternatively, you can use a single config.json file. See the GitHub repository for detailed configuration examples.

Getting Help

For issues, check: bot permissions, required intents are enabled, log channel is configured, and bot can send messages in the log channel. Visit the GitHub repository for detailed documentation and support.
